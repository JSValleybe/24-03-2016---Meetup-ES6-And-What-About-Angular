What are these components?

  • Creating mini application pieces that are self defined and can be used to compose our application
  • Scalable application architecture
  • Better organization and collaboration
  • Easier re-usability
  • Simpler reasoning about your code.
  • Angular 2 takes this approach as does React, and web components.

Components in Angular 1.X

  • Scalable architecture in AngularJS
  • File Structure (components)
  • Modules and submodules
  • Prefer the 'controller as' syntax
  • ui-router state params for sharing information
  • ... (more at Angular Styleguide)
  • The component approach in Angular: Basically, removing all the standalone controllers that were bound to views and treating directives as first class citizens.
  • ES6/ES2015 makes it easy
  • Proof of concept

Step 2 - Better variables

  • ES2015 introduces two new ways, let and const.
  • Both have local scoping and don't hoist.
  • const is immutable as well.
  • We will use const for everything unless we need to change its value, then we'll go back to let.
  • Change var by const and let

Step 5 - Refactor services

  • Difference between service and factory
  • .service() and .factory() are synonymous. A service is a "newable" function and uses this. Therefore it will always return an object. A factory can return whatever we'd like.
  • Create a function for your service/factory which returns a factory object
  • You could use an ES6 class
  • Implement private and public methods
  • Export public methods
  • Inject dependencies
  • Export service
  • Register service with Angular
angular.module('app', [])
// an actual service
// is a "newable" function so no arrows please
service('myService', function(){
  this.action = ()=>{};
})
// a "service" but technically a factory
.factory('myFactory', ()=> {
  return 'heeeeey';
});

Recap

      categories/

      categories.tmpl.html
  • component.js is our entry file for the component. It creates the angular module and registers the directive to it. Also, if you needed routing, this is where you would do.
  • component.controller.js this is the controller file where we create the controller for our component's directive.
  • component.directive.js this is where we create the directive that will be the meat and potatoes of our component. It also has the responsibility of importing the styles and template
